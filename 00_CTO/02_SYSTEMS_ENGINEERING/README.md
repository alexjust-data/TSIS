# Systems Engineering

## Objetivo

Diseñar TSIS como sistema.

No se centra en IA.

No se centra en trading.

Se centra en arquitectura.

## Preguntas

- ¿Cómo se comunican los módulos?
- ¿Dónde viven los datos?
- ¿Cómo se desacopla el sistema?
- ¿Qué contratos existen?

## Contenido esperado

- Architecture
- Module Contracts
- Data Flow
- Event Driven Design
- Interfaces
- Technical Decisions

## Resultado esperado

Arquitectura mantenible durante años.
